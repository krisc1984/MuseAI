# 会话标题兜底与历史记录改名设计

## 功能概述

修复聊天和旧冒险会话因大模型标题生成失败而无法保存的问题，并允许用户在历史会话弹窗中修改会话名称。改动覆盖桌面端与移动端的保存行为，历史弹窗改名交互用于桌面端共用的 `SessionHistoryModal`。

## 成功标准

- 已有有效标题的会话再次保存时，不调用大模型生成标题。
- 新会话标题生成失败或返回空白时，仍能使用本地标题保存。
- 聊天和旧冒险遵守同一套标题选择规则。
- 用户能在历史聊天和历史冒险弹窗中修改标题。
- 修改当前打开会话的标题后，页面标题同步更新。
- 原有打开、筛选和删除历史会话的行为不受影响。

## 标题选择规则

有效标题需满足：

- 去除首尾空白后不为空。
- 聊天标题不等于默认占位标题 `新聊天`。
- 冒险标题不等于默认占位标题 `新故事`。

保存时依次执行：

1. 当前标题有效时直接沿用，不调用 `summarize_text`。
2. 当前标题无效时调用 `summarize_text`。
3. 模型调用失败或返回空白时，取首条非空用户消息作为标题。
4. 首条用户消息去除首尾空白，最多保留 15 个字符；超出时追加 `...`。
5. 没有可用用户消息时，聊天使用 `未命名会话`，冒险使用 `未命名故事`。
6. 选定标题后继续调用 `save_agent_session`，标题生成失败不得阻断保存。

标题选择逻辑应放入小型共享工具函数，避免桌面聊天、桌面冒险、移动聊天和移动冒险产生不同规则。

## 历史会话改名交互

`SessionHistoryModal` 的每条历史记录增加编辑按钮，位置与删除按钮并列。

- 点击编辑按钮后，该条标题原位切换为输入框。
- 输入框自动聚焦并选中现有标题。
- Enter 或确认按钮提交。
- Escape 或取消按钮放弃修改。
- 去除首尾空白后的空标题禁止提交，并显示中文提示。
- 提交期间禁用该条确认操作，避免重复请求。
- 保存成功后退出编辑状态并刷新历史列表。
- 保存失败时保留编辑状态并显示中文错误提示。
- 编辑操作不得触发打开会话。
- 删除其他会话及筛选行为保持不变。

父页面负责调用现有后端命令：

```text
update_agent_session_title({ id, title })
```

若被修改的会话是当前打开会话，父页面同步更新 store 中的 `sessionTitle` 和对应 ref。

## 视觉方向

- 延续 MuseAI 暖色极简风格和现有弹窗密度。
- 编辑按钮使用 Ant Design 线性图标与文本按钮，不增加新卡片或二次弹窗。
- 输入框占用原标题区域，不改变日期、标签和删除按钮的主要布局。
- 使用现有暖灰文字、浅边界和强调色，交互优先清晰，不增加装饰动画。

## 状态与错误处理

- 默认：显示标题、日期、元数据标签、编辑和删除操作。
- 编辑：显示输入框、确认和取消操作。
- 提交中：确认操作处于加载或禁用状态。
- 空标题：阻止请求并显示 `会话名称不能为空`。
- 请求失败：显示 `修改会话名称失败`，保留用户输入。
- 请求成功：显示 `会话名称已修改`，刷新列表并同步当前标题。

## 测试范围

前端测试至少覆盖：

- 已有有效标题保存时没有调用 `summarize_text`。
- 默认标题保存时调用标题生成。
- 标题生成抛错时使用首条用户消息并成功保存。
- 标题生成返回空白时使用首条用户消息。
- 没有可用用户消息时使用最终兜底标题。
- 历史记录进入编辑、取消、空标题拦截和成功提交。
- 改名请求使用正确的会话 ID 与去除首尾空白后的标题。
- 修改当前会话后页面标题同步。
- 编辑按钮不会打开会话。

验证命令：

```text
npm run test
npx -y react-doctor@latest . --verbose --diff
```

必要时补充 `npm run build`，确认 TypeScript 和生产构建无回归。

## 范围限制

- 不修改会话文件格式。
- 不新增 Rust 命令，复用 `update_agent_session_title`。
- 不重构历史会话弹窗之外的列表组件。
- 不改变归档流程生成建议标题的行为。
- 不为会话名称增加额外长度配置；输入值按现有数据模型保存。

## 自检

- 无待定项或占位内容。
- 桌面端与移动端保存规则一致。
- 默认标题和用户自定义标题有明确区分。
- 改名交互、错误状态和当前会话同步均有确定行为。
