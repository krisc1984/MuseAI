## 排查：AI反向分析大纲 - 思考深度关闭是否生效

**结论**：代码层面无 bug。`thinkingDepth=off` 被正确传递到 Rust 后端，并在 OpenAI API 请求 body 中设置 `enable_thinking: false`。但该参数是非标准 OpenAI 参数，实际效果取决于模型：
- 标准 GPT-4o/GPT-3.5：参数被静默忽略，但这些模型本就没有思考模式
- o系列/DeepSeek-R1：这些模型推理是架构级别的，API 参数可能无法完全关闭
- 流式输出做了防护：只捕获 content，不捕获 reasoning_content
