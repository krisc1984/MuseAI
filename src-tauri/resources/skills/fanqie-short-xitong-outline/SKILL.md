# 番茄短篇系统穿越大纲

## 工作流程

1. 先确认目标产物：`选题方向`、`剧情节奏分析`、`全文大纲`、`开篇钩子`、`分段细纲` 中选择一种；如果用户已经说清楚，就直接执行。
2. 生成选题、节奏分析或大纲前，先阅读 `references/fanqie-short-xitong-patterns.md`，把输出锚定在本地短篇系统穿越样本的结构，而不是泛泛写穿越文。
3. 先定一句话系统规则：主角绑定了什么任务，失败/成功会付出什么代价，系统规则如何反转读者预期。
4. 再定主角清醒目标：回家、摆烂、远离男主、完成反杀、利用系统漏洞、让攻略对象付出代价，只选最能制造爽感的一条主线。
5. 制作大纲时按短篇分段推进，不要写成长篇卷纲；每一段都要有明确情绪功能：规则揭示、反套路、误判、作死/摆烂、证据反击、系统反转、追悔惩罚、余韵。

## 输出规则

- 除非用户另有要求，否则全部使用中文。
- 优先输出可直接写正文的内容，不写空泛理论。
- 本地样本多为非完整全文或片段稿；分析样本时必须标注“样本停留处”，不要把推演写成原文事实。
- 开篇必须在 300 字内抛出强钩子：系统解绑、攻略失败、死亡倒计时、任务反转、系统惩罚对象不是主角、主角穿进虐文现场、主角突然不装了。
- 系统规则必须具体可执行，不只写“绑定系统”：要写任务指标、倒计时、惩罚对象、回归条件、系统漏洞、主系统限制、NPC 觉醒或反攻略任务。
- 主角必须清醒，不要让主角只被剧情拖着走；要让主角主动利用信息差、身份差、系统规则或现实资源改变局面。
- 攻略对象追悔不能只写“后悔了”；要有代价：被抹杀、被公开审判、失去主角、任务反噬、世界崩塌、权力失效、余生记得却无法挽回。
- 不要照搬样本人名、设定、标题、标志性桥段或具体句子，只提取节奏机制。
