# 番茄短篇规则怪谈大纲

## 工作流程

1. 先确认目标产物：`选题方向`、`剧情节奏分析`、`全文大纲`、`开篇钩子`、`分段细纲` 中选择一种；如果用户已经说清楚，就直接执行。
2. 生成选题、节奏分析或大纲前，先阅读 `references/fanqie-short-guize-patterns.md`，把输出锚定在本地短篇规则怪谈样本的结构，而不是泛泛写恐怖故事。
3. 先定一句话核心规则：谁被拉进什么场景，遇到哪条看似简单却会杀人的规则。
4. 再定主角破局优势：民俗规矩、身份关系、NPC职权、弹幕提示、亲友救援、暗恋对象保护、考试系统漏洞，只选最能连续制造反转的一条主线。
5. 制作大纲时按短篇分段推进，不要写成长篇副本卷纲；每一段都要有明确功能：规则出现、试错死人、主角误判、优势显形、规则升级、身份反转、终局破局、余韵钩子。

## 输出规则

- 除非用户另有要求，否则全部使用中文。
- 优先输出可直接写正文的内容，不写空泛理论。
- 开篇必须在 300 字内抛出强钩子：高考不能写名字、宿管是亲戚、鬼王认亲、NPC被玩家绑走、杀人鬼在写作业、民俗规矩压制怪谈等。
- 规则必须具体、可执行、会误导人；不要只写“不要违反规则”，要写清楚时间、动作、对象和陷阱。
- 每 1-2 段必须新增一个规则、一次死亡/惩罚、一次弹幕误判、一次身份反转或一次破局证据。
- 主角不能只靠运气活下来；要让优势能持续推进剧情：懂老规矩、看见弹幕、认识NPC、被NPC保护、熟悉系统职场、拥有关键监护人。
- 恐怖与爽感要交替出现：先让读者怕规则，再让主角用特殊优势打破读者预期。
- 注意样本多为开头片段，不要假装已有完整全文；制作全文大纲时要基于样本节奏合理续推。
- 不要照搬样本人名、设定、标题、标志性桥段或具体句子，只提取节奏机制。
